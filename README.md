# 명언 생성 웹페이지 (Vibe Coding 입문)

`index.html + style.css + app.js` 3파일만으로 만든 **반응형·접근성 준수** “명언 생성 웹페이지”입니다.  
버튼 클릭으로 랜덤 명언(문구+저자)을 표시하고, 사용자가 직접 명언을 추가/삭제하며, 복사·공유·ChatGPT에게 묻기까지 할 수 있습니다.

## 주요 기능

- **랜덤 명언 표시**
  - 기본 20개 명언 + 사용자가 추가한 명언을 합쳐서 랜덤으로 보여줍니다.
  - 표시 시 **페이드 인 애니메이션** 적용(사용자가 `prefers-reduced-motion: reduce`면 애니메이션 비활성).
- **내 명언 추가/삭제 (localStorage)**
  - 문구가 비어 있으면 저장되지 않도록 **간단 유효성 검사**를 합니다.
  - 같은 문구 중복 저장을 간단히 방지합니다.
  - 저장/삭제 내역은 **브라우저의 `localStorage`**에 남아 새로고침해도 유지됩니다.
- **클립보드 복사**
  - `navigator.clipboard` 우선 사용, 실패 시 대체 방식으로 복사 시도합니다.
- **공유**
  - Web Share API(`navigator.share`)가 지원되면 공유 시트가 열립니다.
  - 미지원 환경에서는 **트위터 공유 링크**로 대체합니다.
- **ChatGPT에게 묻기**
  - 현재 명언을 바탕으로 “설명 + 오늘 실천할 행동 3가지”를 요청하는 프롬프트를 만들어 새 탭으로 엽니다.
- **라이트/다크 모드 선택**
  - 헤더의 **테마 버튼**으로 `시스템 → 라이트 → 다크 → 시스템` 순환 전환합니다.
  - 사용자가 선택한 테마는 `localStorage`에 저장되어 유지됩니다.
  - `system` 선택 시 시스템 다크모드 변경도 자동 반영합니다.

## 파일 구조

```
vibecoding_exam_260630/
├─ index.html   # UI 마크업(접근성 포함)
├─ style.css    # 모바일 퍼스트, 다크모드, 애니메이션
└─ app.js       # 랜덤/CRUD/복사/공유/테마 로직
```

## 실행 방법

1. 파일 3개(`index.html`, `style.css`, `app.js`)가 같은 폴더에 있는지 확인합니다.
2. `index.html`을 브라우저에서 열면 바로 동작합니다.

팁: 로컬 파일에서도 대부분 동작하지만, 일부 브라우저에서 클립보드/공유 기능이 제한될 수 있어요. 그 경우 간단한 로컬 서버로 여는 것도 좋습니다.

## 저장 데이터 (localStorage)

- **사용자 명언 목록 키**: `vibe_quotes_v1`
- **테마 설정 키**: `vibe_theme_v1` (`"light" | "dark" | "system"`)

브라우저 개발자도구 → Application/Storage → Local Storage에서 확인/삭제할 수 있습니다.

## 접근성/UX 포인트

- ** 스킵 링크**(본문 바로가기)
- **명언 영역 `aria-live`**로 화면 낭독기 업데이트 지원
- 버튼/입력에 **명확한 레이블**
- `:focus-visible` 포커스 스타일 제공(키보드 사용자 배려)
- 모바일 퍼스트 레이아웃(카드 최대폭, 버튼 그리드 반응형)
